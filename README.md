# Журнал обслуживания бортов

Telegram-бот для учета работ на бортах (дронах). Ведет журнал: кто, когда и что делал с каким бортом.

**Telegram:** @alga_drone_log_bot

## Возможности

- Регистрация сотрудников через Telegram
- Управление бортами (добавление, удаление, восстановление)
- Логирование работ с категориями, описанием, фото и документами
- История по борту, сотруднику, дате, поиск по описанию
- Шаблоны для быстрого логирования типовых работ
- Экспорт в PDF (по борту, за период, полный)
- Статистика по бортам, сотрудникам, категориям
- Автоматические напоминания о необслуживаемых бортах
- Еженедельный дайджест для руководителей
- Автоматический бэкап БД
- Журнал действий (audit log)
- Защита от дубликатов записей
- Уведомления в групповой чат

## Ролевой доступ

Три уровня доступа: работник (worker), руководитель (lead), админ (admin).

| Возможность | worker | lead | admin |
|---|:---:|:---:|:---:|
| Регистрация (/start) | + | + | + |
| Запись работы (/log, /log_tpl) | + | + | + |
| Редактирование своих записей (/edit_log) | + | + | + |
| История своих записей (/history_my) | + | + | + |
| Просмотр истории борта (/history) | только свои | все записи | все записи |
| Видит автора записи (ФИО, должность) | нет | да | да |
| История по дате (/history_date) | только свои | все записи | все записи |
| Поиск по описанию (/search) | только свои | все записи | все записи |
| Экспорт по борту (/export) | только свои | полный | полный |
| Добавление бортов (/board_add) | - | + | + |
| Статистика (/stats) | - | + | + |
| Экспорт за период (/export_period) | - | + | + |
| Удаление бортов (/board_delete) | - | - | + |
| Восстановление бортов (/restore_board) | - | - | + |
| Удаление записей (/delete_log) | - | - | + |
| Восстановление записей (/restore_log) | - | - | + |
| Полный экспорт (/export_all) | - | - | + |
| Управление ролями (/set_role) | - | - | + |
| Список сотрудников (/users) | - | - | + |
| Журнал действий (/audit) | - | - | + |
| Управление шаблонами (/template_add, /template_delete) | - | - | + |

Роль назначается админом через `/set_role <telegram_id> <role>`. Начальные админы задаются в `ADMIN_IDS`.

## Стек

- Python 3.12+
- aiogram 3 - Telegram Bot API
- aiosqlite - async SQLite
- pydantic-settings - конфигурация из .env
- structlog - структурированное логирование
- fpdf2 - генерация PDF

## Установка и запуск

### Docker (рекомендуется)

```bash
cp .env.example .env
# Заполнить BOT_TOKEN и ADMIN_IDS в .env

docker compose up -d
```

### Локально

```bash
cp .env.example .env
# Заполнить BOT_TOKEN и ADMIN_IDS в .env

pip install -e .
python -m bot
```

## Конфигурация (.env)

| Переменная | Описание | По умолчанию |
|---|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather | - |
| `ADMIN_IDS` | Telegram ID админов (через запятую) | - |
| `DB_PATH` | Путь к БД SQLite | `./data/drone_log.db` |
| `LOG_LEVEL` | Уровень логирования | `info` |
| `PAGE_SIZE` | Записей на страницу | `5` |
| `NOTIFY_CHAT_ID` | ID группового чата для уведомлений | - |
| `REMINDER_DAYS` | Дней без обслуживания для напоминания | `14` |
| `BACKUP_ENABLED` | Автоматический бэкап БД | `true` |
| `DIGEST_ENABLED` | Еженедельный дайджест | `true` |

## Структура проекта

```
src/bot/
  __main__.py        - точка входа
  config.py          - конфигурация (pydantic-settings)
  tasks.py           - фоновые задачи (бэкап, напоминания, дайджест)
  export_pdf.py      - генерация PDF
  db/
    connection.py    - подключение к SQLite
    migrations.py    - схема и миграции
    employees.py     - CRUD сотрудники
    boards.py        - CRUD борта
    work_logs.py     - CRUD записи работ
    templates.py     - CRUD шаблоны
    audit.py         - журнал действий
  handlers/
    start.py         - регистрация
    board.py         - управление бортами
    log.py           - логирование работ (FSM)
    history.py       - просмотр истории
    admin.py         - админ-команды, экспорт, статистика
    template.py      - шаблоны
  states/
    registration.py  - FSM регистрации
    work_log.py      - FSM логирования и редактирования
  keyboards/
    inline.py        - inline-клавиатуры
  middleware/
    auth.py          - авторизация и проверка ролей
```
